    Kernels

    Firecracker requires a kernel image to boot a microVM. This guide explains Firecracker's kernel requirements and how to build a custom kernel for use with Fireactions.

    For the complete support matrix and end-of-support dates, see Firecracker's Kernel Support Policy.

    Required kernel configuration

    To function properly with Firecracker, your kernel must include specific configuration options. The minimum requirements vary based on your boot method and architecture.

    Universal requirements

    All Firecracker kernels must have:

    CONFIG_SERIAL_8250_CONSOLE=y    # Serial console support
CONFIG_PRINTK=y                 # Kernel logging
CONFIG_BLK_DEV_INITRD=y         # Initial RAM disk support
CONFIG_VIRTIO_MMIO=y            # VirtIO MMIO device support
CONFIG_VIRTIO_BLK=y             # VirtIO block devices
CONFIG_VIRTIO_NET=y             # VirtIO network devices
CONFIG_VIRTIO_BALLOON=y         # VirtIO memory balloon
CONFIG_VIRTIO_VSOCKETS=y        # VirtIO vsock support

    Architecture requirements

    x86_64: 

    CONFIG_KVM_GUEST=y              # Enables KVM clock for timekeeping
CONFIG_SERIO_I8042=y            # i8042 serial I/O support for shutdown
CONFIG_KEYBOARD_ATKBD=y         # AT keyboard support for shutdown

    ARM (aarch64): 

    CONFIG_ARM_AMBA=y               # AMBA bus support
CONFIG_SERIAL_OF_PLATFORM=y     # Platform serial support
CONFIG_RTC_DRV_PL031=y          # RTC driver for timekeeping

    Optional requirements for boot methods

    ACPI Boot (x86_64 only): 

    CONFIG_ACPI=y
CONFIG_PCI=y

    PCI Boot: 

    CONFIG_PCI=y
CONFIG_VIRTIO_PCI=y
CONFIG_PCI_MSI=y
CONFIG_PCIEPORTBUS=y

    Building a custom kernel

    You need to build your own kernel image from one of the provided configurations. Here's how to do it:

    Prerequisites

    Install the required build tools:

    # Debian/Ubuntu
sudo apt-get update
sudo apt-get install -y build-essential libncurses-dev bison flex libssl-dev libelf-dev bc

# RHEL/CentOS/Amazon Linux
sudo yum groupinstall "Development Tools"
sudo yum install -y ncurses-devel bison flex elfutils-libelf-devel openssl-devel bc

    Building from an existing kernel configuration

    Download the Linux kernel source matching the configuration version:

    # For 6.1 kernel
wget https://cdn.kernel.org/pub/linux/kernel/v6.x/linux-6.1.102.tar.xz
tar -xf linux-6.1.102.tar.xz
cd linux-6.1.102

    Copy the Fireactions kernel configuration to .config in the kernel source directory. Alternatively, you can use your own configuration.

    (Optional) Customize the configuration:

    make menuconfig  # Interactive configuration
# or
make olddefconfig   # Use defaults for new options

    Build the kernel:

    # Use all available CPU cores for faster compilation
make -j$(nproc) vmlinux

    The resulting kernel image will be at vmlinux in the kernel source directory.

    Update your Fireactions configuration to use the new kernel:

    pools:
  - name: 2vcpu-4gb
    kernel_image_path: /var/lib/firecracker/kernels/6.1/vmlinux
    # ... other settings

    Troubleshooting

    Build Fails with Missing Dependencies

    Ensure all build dependencies are installed. The kernel build system will report missing dependencies during the configuration or build phase.

    Kernel Doesn't Boot

    • Verify you're using an uncompressed vmlinux image, not bzImage
    • Check that required configuration options are enabled (see Required Kernel Configuration above)
    • Review Firecracker logs for specific error messages
    • Ensure kernel version is supported by your Firecracker version
